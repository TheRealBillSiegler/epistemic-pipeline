# Devon — ML engineer evaluating the formalism

- **Age/role:** 41, staff ML engineer; builds LLM eval harnesses; deeply skeptical of "trust me" AI tools.
- **Tech comfort:** Expert. Reads source before docs. Judges a repo by its tests and its README's first 30 lines.
- **Use frequency:** One deep evaluation session; adopts only if the core claim (auditable belief revision) checks out in code.
- **Accessibility needs:** None.
- **Primary goal:** Verify the pitch — pure R, replayable traces, honest uncertainty — by running it: clone, test, drive the library, replay a trace, try to break determinism.
- **Must-haves:** Working example he can run in under 5 minutes; type hints; the math traceable to literature; an obvious extension point for plugging in a real LLM.
- **Deal-breakers:** Claims in docs the code doesn't back; hidden nondeterminism; mock-only integrations presented as features.
- **Frustration style:** Files precise, slightly caustic issues. Respects projects that state their own limits.
