# Sam — blind developer, screen-reader user

- **Age/role:** 37, backend developer; blind since birth; NVDA + Firefox, terminal-first workflows.
- **Tech comfort:** Expert developer; evaluates tools by how well they work without a mouse or vision.
- **Use frequency:** Occasional — evaluating whether the tool (and its docs) are usable enough to recommend to his team.
- **Accessibility needs:** Screen reader (NVDA). Needs: semantic headings, alt text or text equivalents for diagrams, keyboard-navigable UI, CLI output that reads linearly, math that isn't image-only.
- **Primary goal:** Read the docs site end-to-end with a screen reader, drive the CLI and library from the terminal, and hit no information that exists only in a picture.
- **Must-haves:** Every Mermaid diagram's content also stated in prose; tables that linearize sensibly; skip-to-content; MathJax (accessible) rather than math images.
- **Deal-breakers:** Diagrams as the sole carrier of an explanation; unlabeled UI controls in the future #9 interface.
- **Frustration style:** Methodical; documents exactly where the experience breaks and what a fix costs.
