# Evidence-Based Subjective Logic (arXiv:1402.3319)

**Venue / access:** arXiv. Full text read via ar5iv HTML rendering (ar5iv.labs.arxiv.org/html/1402.3319).
**Source:** https://arxiv.org/abs/1402.3319

## What it is about

Standard Subjective Logic (SL) handles uncertainty well but breaks when applied to trust networks. Its discounting operator discards evidence and gives results that depend on how the network is drawn, not just what agents believe. This paper introduces Evidence-Based Subjective Logic (EBSL), which replaces SL's discounting operator with one defined entirely in terms of evidence flow. The new operator satisfies right-distributivity, which the old one does not. The result is an algebra where reputation can be computed iteratively over any trust network — arbitrary topology, no information thrown away.

## Key topics and concepts

- **Subjective Logic (SL):** A framework that represents beliefs as opinion tuples `(b, d, u)` — belief, disbelief, uncertainty — constrained to sum to 1.
- **Opinion-evidence mapping:** Each opinion maps to a positive/negative evidence pair `(p, n)` via `(b, d, u) = (p, n, c) / (p + n + c)`, where `c` is a free scaling constant.
- **Consensus operator (⊕):** Fuses two independent opinions by pooling their evidence: `x ⊕ y` corresponds to evidence `(p1 + p2, n1 + n2)`.
- **Scalar multiplication (·):** Scales evidence linearly — `α · x` multiplies both `p` and `n` by `α`, so `p(α · x) = αp(x)` and `n(α · x) = αn(x)`.
- **EBSL discounting operator (⊠):** Definition 12 is generic — `x ⊠ y = g(x) · y` where `g: Ω′ → [0,1]` is a designer-chosen function the paper deliberately leaves free. One example it offers is `g(x) = x_b` (which, since `x_b + x_d + x_u = 1`, reduces to using x's belief mass as the scaling factor on y's evidence).
- **Right-distributivity:** `x ⊠ (y ⊕ z) = (x ⊠ y) ⊕ (x ⊠ z)` — the key property EBSL satisfies and SL's ⊗ does not.
- **Flow-based reputation:** Systems like EigenTrust and PageRank that aggregate trust automatically across a graph but cannot express uncertainty.
- **Double-counting problem:** SL's ⊗ operator can count the same piece of evidence multiple times when evidence flows through multiple paths.
- **Non-dogmatic opinions:** Opinions restricted to `u > 0`, denoted Ω′, so no agent claims absolute certainty.

## Main claims and findings

- SL's opinion discounting operator ⊗ has three pathological properties: (1) it is not right-distributive over ⊕, (2) it can make evidence from highly-certain opinions disappear — Example 2 shows `p(x ⊗ y) ≈ p(x)/4` even when y contains effectively infinite evidence — and (3) reputation computation depends on the network's graph structure.
- **Theorem 5.1:** The evidence-opinion mapping `(b, d, u) = (p, n, c)/(p + n + c)` follows from four minimal requirements alone — ratio preservation, normalization, zero-evidence behavior, and convergence to certainty with unlimited evidence — independently of any Beta distribution assumption.
- **Lemma 1:** The consensus rule (⊕) emerges from the general mapping formula regardless of the value of the scaling constant `c` (not only for `c = 2` as in standard SL).
- **Lemma 2 (scalar multiplication properties):** `p(α · x) = αp(x)` and `n(α · x) = αn(x)`; for natural number `n`, `n · x = x ⊕ x ⊕ … ⊕ x` (n times).
- **Lemma 3 (distributivity):** `α · (x ⊕ y) = (α · x) ⊕ (α · y)` — scalar multiplication distributes over consensus.
- **Definition 11 (scalar multiplication formula):** `α · x = (αx_b, αx_d, x_u) / (α(x_b + x_d) + x_u)`.
- **EBSL discounting formula (Definition 12):** `x ⊠ y = g(x) · y` with `g: Ω′ → [0,1]` free — the paper "postpone[s] the precise details of how the function g can/should be chosen." The choice `g(x) = x_b` is given as one instance, scaling all of y's evidence by x's belief mass.
- **Reputation algorithm:** Solves `r_{i,j} = A_{i,j} ⊕ ⊕_{k≠i} (r_{i,k} ⊠ A_{k,j})` iteratively, where `A` is the direct-trust matrix. This works on arbitrary network topologies without discarding edges or fixing a canonical traversal order.
- The EBSL algebra achieves the stated goal: flow-based automation (no graph-structure dependence) combined with consistent uncertainty handling (no information loss, no double-counting).
